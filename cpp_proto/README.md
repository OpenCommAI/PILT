# cpp_proto — 4 protocols, C++ library + bench + dashboard

C++17 implementations of `PILT` / `DCTCP` / `LTP` / `PLOT`, packaged
with:

- 4 ctest binaries (wire / encoder / end-to-end / dctcp+ltp+plot round-trip)
- POSIX TCP demo (PILT worker / parameter server)
- head-to-head benchmark (synthetic gradients → all 4 protocols → JSON)
- single-file interactive Plotly dashboard

No external C++ dependencies. Only requires `CMake ≥ 3.16`,
`GCC/Clang C++17`, `pthread`, and `Python 3 (numpy)` for
`viz/sim_to_json.py`.

---

## Quick start

```bash
cd cpp_proto
bash run_demo.sh
# → open http://localhost:8080/viz/dashboard.html in a browser
```

`run_demo.sh` performs:

1. `cmake -B build -DCMAKE_BUILD_TYPE=Release`, `make -j`
2. `ctest --output-on-failure` (4 tests must pass)
3. Run `compare_protocols`, write `viz/proto_bench.json`
4. `viz/sim_to_json.py` merges `../results/metrics_*.npz` and the bench
   JSON into `viz/dashboard_data.json`
5. `python3 -m http.server 8080`

## Protocol APIs

### `pilt::Encoder` / `pilt::Aggregator`

```cpp
#include "pilt/encoder.hpp"
#include "pilt/aggregator.hpp"

pilt::EncoderConfig cfg{.E_total=0.5f, .eps_min=0.05f, .beta=0.9f, .d=0.05f};
pilt::PILTEncoder enc({S0, S1, S2}, cfg);
auto eps   = enc.compute_ratios();                      // ranked ε_l
auto bytes = enc.encode(round, worker_id, grads);       // top-|g| + residual + CRC

pilt::PILTAggregator ps(K, {S0, S1, S2});
ps.begin_round(round);
for (auto& b : worker_bytes) ps.add(b);
auto avg = ps.finalize_average();                       // weighted by receive count
enc.update_importance(avg);                             // EMA update
```

Wire frame (see `include/pilt/wire.hpp`):

```
magic="PLT1" | version | worker | round | num_layers | payload_len
| layer_id | reserved | layer_size | k | indices[k] | values[k] | ...
| crc32
```

### `dctcp::Encoder` / `dctcp::Aggregator`

```cpp
#include "dctcp/protocol.hpp"
dctcp::Encoder enc(layer_sizes);
auto bytes = enc.encode(round, worker_id, grads);       // dense full-grad

dctcp::Aggregator ps(K, layer_sizes);
ps.begin_round(round);
for (auto& b : worker_bytes) ps.add(b);
auto avg = ps.finalize_average();
```

Application layer is just dense gradient → bytes; ECN/cwnd is handled
by the kernel. `proto::tcp::try_set_dctcp(fd)` attempts
`TCP_CONGESTION=dctcp` on Linux.

### `ltp::Encoder` / `ltp::Aggregator`

```cpp
#include "ltp/protocol.hpp"
ltp::Encoder enc(layer_sizes);
auto bytes = enc.encode(round, worker_id, grads);

ltp::EarlyCloseConfig ec{.min_percent=0.5f, .seed=0xC0FFEE};
ltp::Aggregator ps(K, layer_sizes, ec);
ps.begin_round(round);
ps.add_with_loss(bytes, /*arrived_fraction=*/0.6f);     // triggers bubble-fill
auto avg = ps.finalize_average();

// Stand-alone Early-Close decision:
auto d = ltp::decide_close(at_lt, at_deadline, full_completion_or_neg, ec);
// d.reason ∈ {Full, Early, Deadline}; d.arrived ∈ [0,1]
```

### `plot::Encoder` / `plot::Aggregator`

Two-pass send:

```cpp
#include "plot/protocol.hpp"
plot::LayerLttConfig cfg{.default_ltt=0.7f, .seed=0xBADC0DE};
plot::Encoder enc(layer_sizes);
plot::Aggregator ps(K, layer_sizes, cfg);
ps.begin_round(round);

// First pass
for (k…) ps.add_first_pass(enc.encode(round, k, g_k), per_layer_arrived_k);

// Decide which (worker, layer) pairs need retransmission
auto plan = ps.plan_retx();

// Second pass (only layers below their LTT)
for (k that needs retx) {
    auto retx = enc.encode_retx(round, k, g_k, plan.retx_mask[k]);
    ps.add_retx_pass(retx, per_layer_arrived_2);
}
auto avg = ps.finalize_average();
```

## Head-to-head benchmark

```bash
./build/compare_protocols \
    --K 10 --rounds 30 \
    --layers 1024,4096,16384,2048 \
    --loss_dctcp 0.0 --loss_ltp 0.45 --loss_plot 0.45 \
    --ltt 0.7 --pilt_e_total 0.5 \
    --out viz/proto_bench.json
```

JSON fields:

| key | meaning |
|---|---|
| `bytes_sent` | total bytes the round-trip would put on the wire |
| `delivery_correlation` | Pearson correlation vs the lossless mean |
| `coverage_pct` | fraction of non-zero positions in the aggregate |
| `encode_ms` / `aggregate_ms` | per-round encode / aggregate time |

Reference (default flags, 30 rounds, K=10):

| protocol | bytes_sent | delivery_correlation | encode_ms |
|---|---:|---:|---:|
| DCTCP | 28.3 MB | 1.000 |  7.4 |
| LTP   | 28.3 MB | 0.896 | 41.7 |
| PLOT  | 56.5 MB | 1.000 |  9.9 |
| PILT  | 0.81 MB | 0.859 |  9.0 |

## Dashboard

`viz/dashboard.html` is a single file using Plotly via CDN. Eight panels:

1. Accuracy vs simulated time (4 protocols overlaid)
2. Test-loss curves
3. Same-time comparison table (12 evenly-spaced checkpoints)
4. Time-to-target bar chart
5. C++ bench: bytes_sent
6. C++ bench: delivery_correlation vs bytes_sent (scatter)
7. PILT GA overhead, intrinsic vs exposed
8. Per-round wall time

If you open the file with a `file://` URL the browser will refuse to
fetch the JSON — the dashboard provides a local file upload as a
fallback.
